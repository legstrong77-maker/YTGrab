運作原理

YTGrab 完全在你的電腦本機運行，不經過任何第三方伺服器。以下是完整的技術流程：

1

解析影片資訊

當你貼上 YouTube 網址後，後端使用 yt-dlp 的 --dump-json 指令向 YouTube 發出請求，解析出影片標題、縮圖、可用畫質、時長等完整 metadata。

Shell yt-dlp --dump-json --no-playlist "https://youtube.com/watch?v=..."
2

選擇格式與畫質

YouTube 對每部影片儲存了多個串流：影像軌（不同解析度）與音訊軌是分開儲存的。你選擇的畫質會對應到一個影像格式 ID，系統會自動搭配最佳音訊軌。

YouTube 的 1080p 以上影片，影像與音訊永遠是分開的 adaptive stream
3

下載串流

yt-dlp 分別下載選定的影像軌與音訊軌。下載過程透過 WebSocket 即時回傳進度百分比、下載速度與預估剩餘時間到你的瀏覽器。

Shell yt-dlp -f "bestvideo[height<=1080]+bestaudio" --merge-output-format mp4 ...
4

合併 / 轉換

下載完成後，FFmpeg 負責將影像軌與音訊軌無損合併（mux）為單一 MP4 檔案。若選擇 MP3 模式，則由 FFmpeg 將音訊從 Opus/AAC 轉碼為 MP3 格式（320kbps VBR）。

合併過程不會重新編碼影像，所以速度非常快且不損畫質
5

檔案回傳

合併後的檔案儲存在本機 downloads/ 資料夾，瀏覽器透過 Express 靜態檔案服務取得下載連結。點擊「儲存檔案」即可將檔案另存到你想要的位置。

技術堆疊

yt-dlp
yt-dlp 開源影片下載引擎，支援數千個影音網站，持續更新以繞過平台限制
FF
FFmpeg 業界標準的多媒體處理框架，負責影音串流的合併、轉碼與封裝
JS
Node.js + Express 輕量級 Web 伺服器，處理 API 請求並透過 Socket.IO 提供即時通訊
WS
WebSocket 全雙工通訊協定，實現下載進度的毫秒級即時推送
